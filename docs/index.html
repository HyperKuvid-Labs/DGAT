How it works

From source to graph in one command

DGAT handles everything — parsing, fingerprinting, LLM description, and graph construction — in a single pass.

— 01

Scan & Fingerprint

Walks the directory tree, fingerprints every file with XXH3-128, and respects .gitignore and .dgatignore patterns.

tree-sitter + regex
— 02

Describe with LLM

Sends each file and each import relationship to a local vLLM instance. Every node and edge gets a plain-English description. No cloud, no API keys.

vLLM · OpenAI-compat
— 03

Visualize & Explore

Serves the graph through a three-panel UI: file explorer, interactive WebGL graph with Sigma.js, and an inspector showing dependencies and descriptions.

Next.js · Sigma.js
Features

Everything you need to understand a codebase

Multi-language extraction

Tree-sitter grammars for precision, regex fallback for everything else.

C++TypeScriptPythonGoRustJavaC#

LLM-annotated graph

Every file and every dependency edge gets a natural-language description generated locally by Qwen3.5-2B.

Project blueprint

Synthesises a dgat_blueprint.md — a bottom-up architectural overview built from all file descriptions.

Incremental updates

dgat update re-describes only files whose XXH3 fingerprint changed. Large codebases stay fast.

Static export

Embed the entire graph into a single self-contained HTML file. Share with anyone — no server, no build step.

Live UI

Auto-refreshes every 30 seconds. File explorer, blueprint / graph tabs, and a click-through inspector.

Get started

Not on npm. Not on Homebrew. Clone it.

DGAT isn't a package yet — it's a project you build and run yourself. Three steps to get going.

01

Clone & build

git clone https://github.com/pradheeppr/dgat
cd dgat
cmake -B build && cmake --build build -j$(nproc)
# or: bash install.sh
02

Start vLLM

# any OpenAI-compatible endpoint works
vllm serve Qwen/Qwen2.5-3B-Instruct --port 8000
03

Run on your project

./build/dgat /path/to/your/project
# then open the frontend:
cd frontend && bun dev
# → http://localhost:3000
